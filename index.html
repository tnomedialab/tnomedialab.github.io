Immersive Media

Panoramic Tiled Streaming

Live streaming of panoramic video is challenging. Back in 2010 within the FP7 FascinatE project, TNO pioneered the stream-what-you-watch concept creating the tiled streaming technology. By dividing the panorama into spatial tiles and streaming only those in the current Region-of-Interest of the user, a significant amount of bandwidth can be saved.

  • FascinatE: interactive UHDTV with iCombine
  • FP7 FascinatE project
  • FP7 EXPERIMEDIA iCaCot
  • Commonwealth Games 2014 with BBC

360 video Tiled Streaming

In the early 2010's, the interest in 360 videos begun skyrocketing. Strong of our experience with panoramic tiled streaming, we created the next evolution of Tiled Streaming that uses only one decoder by leveraging existing features of the HEVC video codec.

  • First Public Virtual Reality Demo With Native UHD Resolution, 2016
  • MOS2S: Media Orchestration - Sensor to Screen

Media Centric Networking

MPEG DASH SAND

a.k.a. ISO/IEC 23009 DASH Part 5 Server And Network-assisted DASH.SAND defines a list of messages a DASH client and a SAND-enabled HTTP server (DANE) can exchange. It also defines protocols for a DANE and a DASH client to exchange the SAND messages.

  • SAND QoE metrics reporting over WebSockets Demo
  • Enhancing MPEG dash performance via server and network assistance, 2015  
  • Applications and deployments of server and network assisted DASH (SAND), 2016

Adaptive streaming and SDN/NFV

New network paradigms like SDN and NFV promise to make the network adaptable to application requirements. But how will network adaptation work in combination with the adaptation logic of current streaming applications? TNO is investigating this aspect together with parners in the H2020 TRIANGLE project.

  • TNO's extension to the TRIANGLE testbed

Information Centric Networking

In the context of Information-Centric Networking (ICN), we look at methods to more efficiently request and deliver content to applications, through innovative caching algorithms and advanced data naming approaches. Our applications include: adaptive bitrate video streaming and the Internet of Things.

  • Extending Named Data Networking with support for Names with Range Indications, 2016
  • BidCache: Auction-Based In-Network Caching in ICN, 2016

Standardization

MPEG

The ISO working group JTC 1/SC 29/WG 11 is better known as MPEG. We are active since 2013 by contributing and coordinating work in this standardization body. Below are examples of specifications for which we are editors or main contributors.

  • ISO/IEC 23000-20, Multimedia application format (MPEG-A) -- Part 20: Omnidirectional media application format (OMAF)
  • ISO/IEC 23001-13, MPEG systems technologies -- Part 13: Media orchestration
  • ISO/IEC 23009-5:2017, DASH -- Part 5: Server and network assisted DASH (SAND)
  • ISO/IEC 23009-1:2014/Amd 2:2015, DASH -- Part 1/Amd2: Spatial relationship description (SRD)

DASH-IF

Media streaming is key technology for our work. Joining the DASH Industry Forum (DASH-IF) allows us to be on the forefront of industry needs for OTT technologies. Inside the Interoperability Task Force, we are contributing to this forum. Below are examples of documents reflecting our participation.

  • DASH-IF Implementation Guidelines: Token-based Access Control for DASH (TAC) v1.0
  • DASH-IF position Paper: DASH-IF Position Paper: Server and Network Assisted DASH (SAND)

IETF

We are active in various groups of the IETF and IRTF, most notably CDNI and ICNRG, where we work towards the CDN Interconnection standard and the harmonization of different ICN architectures, respectively.

  • URI Signing for CDN Interconnection (CDNI), Internet-Draft, May 2017
  • Information-Centric Networking: Terminology, Draft, 2017
  • Content Delivery Network Interconnection (CDNI) Request Routing: Footprint and Capabilities Semantics, RFC, 2016
  • Request Routing Redirection Interface for Content Delivery Network (CDN) Interconnection, RFC, 2016

HbbTV

The HbbTV forum aims at harmonizing the broadcast and broadband delivery of entertainment to consumers through Connected TV’s and set-top-boxes. TNO has been an active member since 2012 and board member since 2014. Our contributions comprise chairing the IPTV Task Force and driving the Application Discovery and Operator Applications work items.

  • Hybrid Broadcast Broadband TV 2, 2016
  • Hybrid Broadcast Broadband TV - Application Discovery over Broadband, 2016
  • Operator Application Specification (under development), 2016

SAND QoE metrics reporting over WebSockets

Each DASH player started from this demo page downloads segments from a media server and at the same time connects to a same DANE. The DASH metrics generated by each DASH client are then sent over a WebSocket connection to the DANE. This way, a service provider, a CDN, can detect issues in the delivery of DASH content and react as quickly as possible to remediate.

  • Code repository
